# docs/

Project documentation for Autodidact.

## Purpose

This folder is the single source of truth for all non-code documentation: product decisions, architecture, infrastructure, and development plans.

Code-level documentation (README files in service directories) lives alongside the code it describes.

---

## What's here

| File / Folder | Purpose |
|---|---|
| [product.md](product.md) | Product vision, teaching model, core user flows |
| [roadmap.md](roadmap.md) | Phase-by-phase feature roadmap |
| [stack.md](stack.md) | Technology choices and rationale |
| [architecture/](architecture/) | System design: overview, data model, infrastructure, C4 diagrams, ADRs |
| [superpowers/](superpowers/) | Implementation plans and design specs |
| [templates/](templates/) | Templates for `CLAUDE.md` and `README.md` files |

---

## Documentation maintenance

Update this README when a new top-level document or folder is added.

If a doc becomes stale, fix it or delete the stale section — stale docs are worse than no docs.
