# JWKS-Based Auth Verification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `supabase.auth.getUser()` (network call per request) with local JWKS-based JWT verification using `jose`, removing the need for `SUPABASE_JWT_SECRET` and `SUPABASE_SECRET_KEY` in the auth path.

**Architecture:** `SupabaseAuthProvider` creates a cached `RemoteJWKSet` from `${supabaseUrl}/auth/v1/.well-known/jwks.json` on construction, then verifies each token locally with `jwtVerify`. The Supabase admin client (`packages/db`) continues to use `SUPABASE_SECRET_KEY` for DB operations — that path is untouched. The factory's `createAuthProvider` no longer requires or accepts a secret key.

**Tech Stack:** `jose` (JWKS + JWT verification), Vitest (tests), TypeScript

## Task 3: Implement JWKS-based `SupabaseAuthProvider`

**Files:**
- Rewrite: `packages/providers/src/implementations/auth/supabase-auth.provider.ts`

- [ ] **Step 1: Replace the implementation**

```typescript
import { createRemoteJWKSet, jwtVerify } from 'jose';
import type { IAuthProvider } from '../../interfaces/auth.js';
import type { AuthUser } from '@autodidact/types';

export class SupabaseAuthProvider implements IAuthProvider {
  private readonly jwks: ReturnType<typeof createRemoteJWKSet>;

  constructor(config: { supabaseUrl: string }) {
    this.jwks = createRemoteJWKSet(
      new URL(`${config.supabaseUrl}/auth/v1/.well-known/jwks.json`),
    );
  }

  async verifyToken(token: string): Promise<AuthUser> {
    let payload: Awaited<ReturnType<typeof jwtVerify>>['payload'];
    try {
      ({ payload } = await jwtVerify(token, this.jwks));
    } catch {
      throw new Error('Invalid token');
    }
    const sub = payload.sub;
    if (!sub) throw new Error('Invalid token');
    const email = typeof payload['email'] === 'string' ? payload['email'] : '';
    return { id: sub, supabaseId: sub, email };
  }
}
```
