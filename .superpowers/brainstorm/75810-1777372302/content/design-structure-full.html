design/README.md — structure

# design/

Single source of truth for all visual values.
Nothing outside this folder defines colors, spacing, or type sizes.
The old `constants/colors.ts` is deleted — all values live here.

## How it fits together
tokens → themes → typography → config
Each layer depends only on the one before it.

**tokens.ts** — primitive scales. Raw color hex values, numeric spacing
  steps, border radius values, icon sizes. No semantic meaning.
  If you need a new color in the app, it starts here.

**themes.ts** — semantic mapping. Names a subset of token values by
  intent: bg, surface, surfaceHover, text, textMuted, border,
  primary, primaryHover, success, warning, danger, overlay.
  Screens and components reference these names, never raw tokens.
  Adding a light theme = add one new object here, zero screen changes.

**typography.ts** — font system. Defines the system font family and
  the named type scale: h1, h2, xl, lg, md, sm, xs.
  Also defines weight tokens: regular, semibold, bold.
  Consumed by config.ts via Tamagui's createFont.

**config.ts** — Tamagui assembly. Calls createTamagui with tokens,
  themes, and fonts. The single export consumed by TamaguiProvider
  in app/_layout.tsx. Nothing else should call createTamagui.
