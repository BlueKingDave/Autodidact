Section 2 — Component Layer (revised)

Updated APIs for AppText, Button, Input, Card.

AppText — replaces Body

Variant sets semantic defaults (color + size + weight). Size and weight props override when needed.

// Variants (semantic defaults)
variant="body"     → size=md, weight=regular, color=text
variant="muted"    → size=md, weight=regular, color=textMuted
variant="caption"  → size=sm, weight=regular, color=textMuted
variant="label"    → size=xs, weight=semibold, color=textMuted, uppercase
variant="error"    → size=sm, weight=regular, color=danger

// Override when needed
<AppText variant="muted" size="lg" weight="semibold">
<AppText variant="body" size="xl">

// Heading kept separate (h1/h2 — structural, not inline)
<Heading size="h1">What do you want to learn?</Heading>
<Heading size="h2">My Courses</Heading>

Button — with size

variant="primary"   → bg=primary, text=white            (Start Learning, Sign In)
variant="danger"    → bg=danger,  text=white            (Sign Out)
variant="ghost"     → bg=transparent, border=border     (secondary actions)

size="sm"           → compact, inline use
size="md"           → default
size="lg"           → full-width CTA

loading={bool}      → spinner replaces label, disabled
disabled={bool}

// Examples
<Button variant="primary" size="lg" loading={isPending}>Start Learning</Button>
<Button variant="danger"  size="lg">Sign Out</Button>
<Button variant="ghost"   size="sm">Cancel</Button>

Input — with label / error / helper

Compound component: wraps label + field + feedback text in a single unit.

label="Email"                     // AppText variant="label" above field
error="Invalid email address"     // AppText variant="error" below (overrides helper)
helper="We'll never share this"   // AppText variant="caption" below

// Border turns danger color when error is set
// Label + error/helper are optional — bare Input still works for chat

// Examples
<Input label="Email" keyboardType="email-address" error={errors.email} />
<Input label="Password" secureTextEntry helper="At least 8 characters" />
<Input multiline maxLength={4000} />  // chat input — no label

Card — with variants

variant="default"   → bg=surface, border              (course cards, info card)
variant="elevated"  → bg=surfaceHover, no border      (highlighted / active)
variant="ghost"     → bg=transparent, border=dashed   (empty state placeholder)

onPress={fn}        → makes card pressable + adds press opacity
disabled={bool}     → opacity=0.45, non-interactive   (locked modules)

// Examples
<Card variant="default" onPress={() => router.push(...)}>
  ...course content
</Card>

<Card variant="default" disabled={isLocked}>
  ...locked module
</Card>

<Card variant="elevated">
  ...profile info
</Card>
