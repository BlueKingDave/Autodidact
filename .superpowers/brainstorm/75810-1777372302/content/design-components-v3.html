README.md — structure

# components/

All shared UI components. Built on Tamagui primitives.
Import from the barrel: `import { Card, AppText } from '@/components'`

## layout/
Components that control page structure and safe area behaviour.

**Screen** — wraps every screen. Handles SafeAreaView, background colour,
and optional ScrollView. Exists so screens don't repeat safe-area boilerplate.

## typography/
Single source of truth for all text rendering.

**Heading** — h1/h2 only. Structural headings, not inline emphasis.
**AppText** — all other text. variant prop is the primary API:
  body, muted, caption, label, error.
  Size and weight props override defaults when needed.

## interactive/
Components the user directly acts on.

**Button** — primary/danger/ghost variants, sm/md/lg sizes, built-in loading state.
**IconButton** — circular icon-only button. Used for the chat send action.
**Input** — text field with colocated label, error, and helper text.
  Exists so label+field+validation are always kept together.
**Chip** — single-select toggle. Used for the difficulty picker on HomeScreen.

## display/
Read-only visual components that present data.

**Card** — surface container. default/elevated/ghost variants.
  onPress makes it pressable; disabled reduces opacity (locked modules).
**Badge** — small status pill. Communicates state at a glance.
**ProgressBar** — linear 0–1 fill bar. Used for course completion.
**PositionBadge** — circular module step indicator. Shows number or ✓.
**ChatBubble** — message bubble. User and assistant variants.
  Renders streaming cursor while isStreaming is true.
**EmptyState** — centred message for lists with no data.

✓ 4 folders — layout, typography, interactive, display
✓ Folder name answers "what kind of thing is this?"
✓ README explains the why for each file, not just the what
✓ Single barrel index.ts — screens always import from one place
